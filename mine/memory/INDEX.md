# mine/memory/ — 个人事件 & 人物记忆库

> 📌 **定位**：保留事实性、细节性的个人经历与人际关系。不归纳压缩，尽量还原原貌。
>
> **与 raw / compiled / explored 的区别**：
> - `raw/` 存的是观点、想法、思考素材
> - `compiled/` 存的是提炼后的主题观点
> - `explored/` 存的是探索中的方法论
> - **`memory/` 存的是「发生过什么、谁是谁」** —— AI 借此快速建立对你的人际/经历背景认知

---

## 🧑 人物档案（people/）

> 每个重要的人一个文件，文件名用 `<姓名或代号>.md`。

| 姓名/代号 | 关系 | 文件 | 备注 |
|---|---|---|---|
| _示例：张三_ | _同事/前同事_ | `people/张三.md` | _共事 3 年_ |
|  |  |  |  |

## 📅 事件档案（events/）

> 按类型分文件夹存放。文件名格式：`YYYY-MM-DD-<事件关键词>.md`。

### 建议分类（按需新建）

- `events/工作/` — 项目、晋升、跳槽、重要会议、职场冲突
- `events/感情/` — 恋爱、婚姻、关系节点
- `events/友情/` — 重要朋友、聚会、断交
- `events/亲情/` — 家人、家庭事件
- `events/爱好/` — 兴趣相关的关键经历
- `events/健康/` — 体检、就医、身体状态变化（偏身体）
- `events/心灵/` — 内心成长、心理状态、自我发现（偏精神/成长）
- `events/财务/` — 重大支出、投资、收入变化
- `events/学习/` — 课程、考试、证书、自学项目
- `events/生活/` — 日常生活点滴、随意记录、偶发性小思考

> 不需要为单条事件提前建空目录，写到对应分类时再建。也可以根据自己的实际情况增删分类。

---

## 🔗 跨链接

- 事件中提到的人，链接到 `people/<人名>.md`
- 由事件引发的思考，写到 `mine/raw/`，并在事件文件末尾互相引用
- 模板：
  - 人物：`people/template.md`（不要直接修改，复制后再填）
  - 事件：`events/template.md`（不要直接修改，复制后再填）
