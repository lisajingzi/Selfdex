# Selfdex

> A mirror of yourself, made of your own thoughts.

---

## 这是什么

Selfdex 是一套用 Markdown + Git 沉淀**自己思想**的目录与流程规范。

它不是笔记软件，不是知识管理工具，也不是 todo 系统。它是一个**数字化的你**——一个由你长期亲手写入、装着你逻辑思维的数据库。任何一个 LLM 读到它，都能用接近你的方式思考、判断、回应。

即使有一天你不在了，这个数据库依然可以让 AI 以你的逻辑继续对话，仿佛你的思维还在。

### 核心作用

- 让任何 LLM 在和你对话前，先认识真正的你
- 让你做决策时能查到「过去的我对这件事怎么想」
- 让你的思想以纯文本形式独立存在，不绑定任何平台、任何模型
- 作为自我观察的一部分，获得旁观者视角，见证你的心路历程与变化


---

## 前置要求（先看这个）

Selfdex 不是自动化工具。**它的全部价值，建立在使用者长期记录自己思考与表达的习惯上**。

如果你不愿意持续地把自己的观点、纠结、判断、反应写下来，Selfdex 帮不了你——因为没有原料，AI 无从认识你，连你自己回头看也只是一个空壳。

适合使用 Selfdex 的人：

- 习惯把想法写成文字（不一定长，但要持续）
- 愿意区分「我自己想的」和「别人说的我认同」
- 接受「AI 只整理、不替你思考」的边界

如果你只想要「自动总结我读过的文章」、「存储一些知识性的内容」，Selfdex 不做这件事。

## 设计哲学

### 1. 沉淀的是思想，不是知识

知识可以从任何地方搬过来；思想只能由你自己产生。
Selfdex 的核心层 `mine/` 只装你自己的东西——你的观点、你的纠结、你的判断、你的反应。看过的文章、听过的播客、AI 说过的话，都不是你的。

### 2. 内化必须经手

外部内容（`others/`）哪怕你再认同，也必须经过一次亲手「整理和贴入」才能进入 `mine/`。这一步是故意设置的认知关卡——它让你区分「我认同它」和「我把它内化为自己的判断」。

ps.通常others我会使用微信读书MCP接入我读书划线内容，按书名分文件夹归纳；你也可以贴进来一些你认可的内容，但记住，一定是要经过你筛选和理解的，你认同的内容，并非不加思考的贴入链接等，这违背了体系的设计的初衷。


### 3. 归档即思考

每一次手动归档都是一次主动的认知校验。Selfdex 拒绝「AI 全自动整理」，每次AI辅助整理后，都需要你逐字阅读确认，当然你也可以选择自己整理。

## 体系结构：双轨 + 三层 + memory

```
Selfdex/
├── mine/                     ← 我的思想（核心）
│   ├── raw/                  原始想法、灵感、对话片段
│   ├── compiled/             同主题碎片聚合形成的主题文档
│   ├── explored/             深度方法论 / 长文产出
│   └── memory/               人与事的档案
│       ├── people/           关系、相处、对某人的判断
│       └── events/           含人名 + 时间的具体事件
│
├── others/                   ← 别人的素材
│   ├── raw/                  文章/书/播客摘录 + 我的反应
│   ├── compiled/             跨来源的主题汇总
│   └── explored/             启发我的部分（待「搬运」到 mine）
│
├── profile.md                ← 「我是谁」画像（喂给 LLM 用）
├── INDEX.md                  ← 跨工作区导航地图
│
└── docs/                     ← 流程与规则（AI 执行前必看）
    ├── flow.md               流程总指挥
    ├── redlines.md           红线门禁
    ├── rules/                各模块执行细则
    ├── templates/            各层模板
    └── user-rules-template.md  贴进 IDE User Rules 的全局指令模板
```

### 双轨：mine vs others

| 轨 | 装什么 | 铁律 |
|---|---|---|
| `mine/` | 自己的想法、观点、判断、反应 | 不被外部内容污染 |
| `others/` | 文章、书、播客、他人观点 | 想内化必须经过「搬运」 |

> ⚠️ **mine 与 others 不互染。**

### 三层：raw → compiled → explored

| 层级 | 含义 | 要求 |
|---|---|---|
| `raw` | 未加工的对话、灵感、摘录 | 快、多，不要求结构 |
| `compiled` | 同主题碎片聚合形成的主题文档 | 有结构，按主题组织 |
| `explored` | 深度方法论 / 长文 | 真正属于自己的产出 |

### memory：人与事的档案

`mine/memory/` 与三层正交，专门处理「具体的人」和「具体的事」：

- `people/<姓名或代号>.md` — 关系背景、互动、你对这个人的判断
- `events/<分类>/` — 含人名 + 时间 + 行为的事件（工作 / 感情 / 友情 / 亲情 / 爱好 / 健康 / 财务 / 学习 / 生活 / ...）

memory 的存在让 AI 在涉及具体人和事的对话中能调出真实背景，而不是只在抽象主题层兜圈子。

## 使用说明

Selfdex 是**半自动协作**：你提供原料与判断，AI 提供整理与起草。

- raw：你直接贴内容，告知AI是mine还是others，AI不加修改保留写入，建议按照时间归档（如按周/月 区分文档）。
- compiled：你发起，告知AI对某一批raw文件进行整理，通常月度整理一次即可，确保有充足的内容，AI会按照raw的内容进行分主题总结归类，你需要进行确认是否准确全面。你的compiled将会以分成不同主题的文件，并被打上tags进入index，后续的对话涉及相关主题将触发读取。
- explored：AI帮助你自我观察的工具，建议至少3个月以上进行一次explore，或许你会得到一些意想不到的观察结果。

小tips：关于对话提炼观点。
- 个人的想法和观点也常见于和AI对话中，若你在IDE中使用Selfdex，建议配置全局rules，你的任意一次对话都可以随时通过指令让AI把整个对话，加入mine-raw-conversation。
- 若你使用网页版LLM对话，建议你vibe coding一个插件以导出某个session中的所有对话，将此md文件加入mine-raw-conversation即可。
- 如果你和我一样经常使用AI对话交流思考，需要此种方式提炼观点，建议建立单独的文件夹保留对话原文，然后再让AI提炼对话观点进入mine-raw。关于raw的rules里面有现成的对话提取方式，不必担心。

## Quick Start

### 1. Clone

```bash
git clone https://github.com/<your-name>/Selfdex.git
cd Selfdex
```

### 2. 写你的 profile

复制 `docs/templates/profile.template.md` → `profile.md`，按模板填。这是 AI 第一次「认识你」的入口。
tips：可以让你常用的LLM帮你总结一份，然后贴进来。

### 3. 配置 IDE 触发词（可选但强烈推荐）

把 [`docs/user-rules-template.md`](docs/user-rules-template.md) 的内容贴进 IDE 的 User Rules / 全局指令。配置后，在任何工作区说「记到 Selfdex」或「翻 Selfdex」，AI 会自动按规则读写。

支持大小写：`记到 Selfdex` / `记到 selfdex` / `存到 Selfdex` / `存到 selfdex` 都识别。

### 4. 开始写

同上方「使用说明」
